# ESP32 Motion Controller

Прошивка управляет шаговым двигателем через драйвер **TMC2160** по интерфейсу **SPI + STEP/DIR**. Экран и энкодер дают два рабочих режима движения и расширенное меню настройки драйвера.

README одновременно является кратким руководством пользователя и справочником по пунктам меню.

## Что умеет прошивка

| Раздел | Назначение |
| --- | --- |
| `Постоянный` | Двигатель вращается непрерывно с заданным направлением и скоростью в оборотах в секунду. |
| `Вибро` | Двигатель ходит туда-сюда между двумя крайними точками с заданной частотой и углом размаха. |
| `Настройка` | Корневое меню настроек TMC2160. Возврат из настроек происходит на тот экран, откуда они были открыты. |
| `TMC2160` | Настройка тока, микрошагов, StealthChop, SpreadCycle/chopper, StallGuard, CoolStep и диагностики. |
| `INA226` | Измерение напряжения и тока через плату INA226 с шунтом `0.01 Ом`; показания выводятся в правом верхнем углу экрана. |
| `/data.db` | Файл настроек в LittleFS. Через кнопки `Сохранить` параметры записываются и применяются после следующей перезагрузки. |

## Управление энкодером

| Действие | Что происходит |
| --- | --- |
| Поворот энкодера | Перемещение по строкам меню. |
| Нажатие на строке-кнопке | Выполнение действия, например переход в экран, сохранение или перезагрузка. |
| Нажатие на числовом поле | Вход/выход из режима редактирования. В режиме редактирования строка подсвечивается инверсией. |
| Поворот в режиме редактирования | Изменение значения с шагом, заданным для поля. |
| Нажатие на переключателе | Переключение `Вкл/Выкл` или одного из двух режимов. |

## Важная логика сохранения

| Кнопка | Что сохраняет |
| --- | --- |
| `Сохранить` на экране `Постоянный` | Направление и скорость постоянного вращения. |
| `Сохранить` на экране `Вибро` | Частоту, угол и текущий микрошаг. |
| `Сохранить` в TMC-разделах | Настройки драйвера TMC2160: режим, ток, микрошаг, интерполяцию и расширенные регистры. |
| `Мотор: Вкл/Выкл` | Применяется сразу, но специально не сохраняется. После перезагрузки мотор не включается неожиданно. |

Настройки читаются после `driver.begin()` и до запуска движения. Это важно: драйвер сначала инициализируется, затем к нему применяются сохраненные регистры.

## Индикатор INA226

В правом верхнем углу всех экранов показывается строка вида `12.34V 1.23A`.

| Поле | Что значит |
| --- | --- |
| `V` | Напряжение шины INA226, `getBusVoltage_V()`. |
| `A` | Ток через шунт `0.01 Ом`, `getCurrent_A()`. |
| `INA --` | Датчик не найден или есть ошибка I2C. Прошивка периодически пробует подключиться снова. |
| `!` после тока | INA226 сообщил overflow, диапазон измерения/шунт/ток нужно проверить. |

Подключение INA226: `SDA = GPIO48`, `SCL = GPIO47`, адрес по умолчанию `0x40`.

## Рабочий экран `Постоянный`

| Поле | Что значит | На что влияет |
| --- | --- | --- |
| `Режим: Постоянный >` | Переход на экран `Вибро`. | Меняет рабочий режим с постоянного вращения на вибрацию. |
| `Мотор: Вкл/Выкл` | Включает или отключает выход драйвера через enable. | При `Выкл` двигатель не должен получать удержание/движение. |
| `Направление: ->/<-` | Направление постоянного вращения. | Меняет знак скорости STEP/DIR. |
| `Скорость об/с` | Скорость вращения в оборотах в секунду. | Пересчитывается в шаги/сек с учетом `200` полных шагов на оборот и текущего микрошагового режима. |
| `Сохранить` | Сохраняет направление и скорость. | После перезагрузки режим `Постоянный` восстановит эти значения. |
| `Настройка` | Переход в корневое меню TMC2160. | Возврат из настроек приведет обратно в `Постоянный`. |
| `Перезагрузка` | Перезапуск ESP32. | Аналог `esp_restart()`. |
| `Очистка EEPROM` | Зарезервированный пункт. | Сейчас не выполняет очистку, код закомментирован. |

Ограничение скорости зависит от микрошагов. Чем выше микрошаг, тем больше STEP-импульсов нужно на один оборот, поэтому максимальная реальная скорость ниже.

## Рабочий экран `Вибро`

| Поле | Что значит | На что влияет |
| --- | --- | --- |
| `Режим: Вибро >` | Переход на экран `Постоянный`. | Меняет рабочий режим с вибрации на постоянное вращение. |
| `Мотор: Вкл/Выкл` | Включает или отключает выход драйвера. | При `Выкл` двигатель не должен двигаться. |
| `Частота` | Частота полного цикла туда-обратно, Гц. | Задает, сколько раз в секунду двигатель должен выполнить полный цикл вибрации. |
| `Угол` | Полный угловой размах между крайними точками. | Если задано `90`, движение идет примерно от `-45` до `+45` градусов относительно центра. |
| `Микрошаг: 1/N` | Текущий микрошаг TMC2160. | Влияет на точность угла и на частоту STEP-импульсов. Значение нормализуется к `1, 2, 4, 8, 16, 32, 64, 128, 256`. |
| `Сохранить` | Сохраняет частоту, угол и микрошаг. | Эти значения будут восстановлены после перезагрузки. |
| `Перезагрузка` | Перезапуск ESP32. | Используется для проверки сохраненных настроек. |
| `Настройка` | Переход в корневое меню TMC2160. | Возврат из настроек приведет обратно в `Вибро`. |

В режиме `Вибро` прошивка считает число микрошагов для заданного угла и чередует цель между двумя крайними точками. При изменении угла, частоты или микрошагов планировщик вибро-движения пересчитывается.

## Корневое меню `Настройка`

| Поле | Что значит |
| --- | --- |
| `<- Назад` | Вернуться на экран, откуда были открыты настройки: `Постоянный` или `Вибро`. |
| `Основные >` | Базовые настройки TMC2160: режим, ток, микрошаг, интерполяция, пороги. |
| `Ток/покой >` | Тонкая настройка рабочего и удерживающего тока. |
| `Chopper >` | Настройка SpreadCycle/classic chopper. |
| `StealthChop >` | Настройка тихого PWM-режима StealthChop2. |
| `Stall/Cool >` | StallGuard2 и CoolStep. |
| `Диагностика >` | Чтение статусов TMC2160 и DIAG-пинов. |
| `Эксперт >` | Низкоуровневые параметры DIAG, защит, силового каскада и gate-driver. |
| `Сохранить TMC` | Сохранить все настройки TMC2160 в `/data.db`. |
| `Перезагрузка` | Перезапуск ESP32. |

## TMC2160: `Основные`

| Поле | Диапазон | Что значит и на что влияет |
| --- | --- | --- |
| `<- Настройки` | Кнопка | Вернуться в корневое меню настроек. |
| `Мотор: Вкл/Выкл` | Переключатель | Немедленно включает или отключает выход драйвера. Не сохраняется специально, чтобы после перезагрузки мотор не включился сам. |
| `Режим: StealthChop/SpreadCycle` | Переключатель | `StealthChop` тише и мягче, хорош для малых/средних скоростей. `SpreadCycle` обычно лучше держит момент и устойчивее под нагрузкой. |
| `Ток RMS` | `100..3100 мА` | Удобная установка рабочего RMS-тока двигателя. Обычно это первый параметр, который стоит настраивать вместо ручного `IRUN`. |
| `Микрошаг: 1/N` | `1..256` | Количество микрошагов на полный шаг. Чем больше микрошаг, тем плавнее движение, но тем выше требуемая частота STEP для той же скорости. |
| `MicroPlyer: Вкл/Выкл` | Переключатель | Включает интерполяцию TMC до 256 микрошагов. Улучшает плавность без изменения количества STEP-импульсов от ESP32. |
| `TPWMTHRS` | `0..1048575` | Порог скорости в сырых единицах `TSTEP` для работы/перехода StealthChop. `0` оставляет поведение максимально простым. |
| `TPOWERDOWN` | `0..255` | Задержка перед снижением тока/переходом в режим покоя после остановки. Чем больше значение, тем дольше держится рабочий режим после остановки. |
| `Fast standstill` | Переключатель | Быстрое поведение при остановке/standstill. Может уменьшить нагрев и удержание, но влияет на ощущение остановки. |
| `Сохранить` | Кнопка | Сохраняет TMC-настройки. |

## TMC2160: `Ток/покой`

| Поле | Диапазон | Что значит и на что влияет |
| --- | --- | --- |
| `IRUN` | `0..31` | Сырая шкала рабочего тока. Чем выше, тем больше ток, момент и нагрев. Обычно удобнее пользоваться `Ток RMS`. |
| `IHOLD` | `0..31` | Сырая шкала удерживающего тока на остановке. Меньшее значение снижает нагрев, но уменьшает удерживающий момент. |
| `IHOLDDELAY` | `0..15` | Плавность/задержка перехода от рабочего тока к удерживающему. |
| `GLOBAL_SCALER` | `0..255` | Глобальный множитель тока. `0` означает дефолт/полную шкалу драйвера; значения `1..31` лучше не использовать; практический диапазон обычно начинается с `32`. |
| `Freewheel` | `0..3` | Поведение двигателя в покое в StealthChop: `0` обычное удержание, `1` свободный ход, `2/3` пассивное торможение через замыкание обмоток. |
| `CS actual` | Только чтение | Текущая фактическая шкала тока, которую сообщает драйвер. Полезно для проверки CoolStep и ограничений. |
| `Сохранить` | Кнопка | Сохраняет TMC-настройки. |

## TMC2160: `Chopper`

| Поле | Диапазон | Что значит и на что влияет |
| --- | --- | --- |
| `TOFF` | `0..15` | Время выключения chopper. Важно: `0` может фактически отключить силовой драйвер, менять осторожно. |
| `TBL` | `0..3` | Blank time, задержка игнорирования измерения тока после переключения ключей. Влияет на стабильность регулирования тока. |
| `HSTRT` | `0..7` | Hysteresis start. Настраивает форму/динамику SpreadCycle. |
| `HEND` | `0..15` | Hysteresis end. Вместе с `HSTRT` влияет на шум, пульсации тока и момент. |
| `CHM: Classic/Spread` | Переключатель | Выбор classic constant-off-time chopper или SpreadCycle. Для обычной работы чаще нужен SpreadCycle. |
| `DEDGE` | Переключатель | Шаг по обоим фронтам сигнала STEP. Может фактически изменить воспринимаемую частоту шагов, включать только если это точно нужно. |
| `VHIGHFS` | Переключатель | Высокоскоростной переход в fullstep. Может помочь на больших скоростях, но снижает плавность. |
| `VHIGHCHM` | Переключатель | Высокоскоростное изменение chopper-режима. Используется для поведения на больших скоростях. |
| `Сохранить` | Кнопка | Сохраняет TMC-настройки. |

## TMC2160: `StealthChop`

| Поле | Диапазон | Что значит и на что влияет |
| --- | --- | --- |
| `PWM autoscale` | Переключатель | Автоматическая подстройка амплитуды PWM в StealthChop. Обычно лучше держать включенным. |
| `PWM autograd` | Переключатель | Автоматическая подстройка градиента PWM. Обычно лучше держать включенным. |
| `PWM_OFS` | `0..255` | Базовое смещение PWM. Влияет на старт, тихий ход и риск пропуска шагов в StealthChop. |
| `PWM_GRAD` | `0..255` | Градиент PWM по скорости/нагрузке. Влияет на то, как StealthChop добавляет энергию при движении. |
| `PWM_FREQ` | `0..3` | Частота PWM StealthChop. Может влиять на слышимый шум и нагрев. |
| `PWM_REG` | `0..15` | Регулятор автоподстройки PWM. Чем выше значение, тем иначе реагирует автоматика StealthChop. |
| `PWM_LIM` | `0..15` | Ограничение изменения PWM. Помогает сдерживать резкие изменения. |
| `PWM scale` | Только чтение | Текущая шкала PWM, которую сообщает драйвер. Полезно смотреть при настройке тихого режима. |
| `Сохранить` | Кнопка | Сохраняет TMC-настройки. |

## TMC2160: `Stall/Cool`

| Поле | Диапазон | Что значит и на что влияет |
| --- | --- | --- |
| `SGT` | `-64..63` | Порог/чувствительность StallGuard2. Подбирается экспериментально под мотор, ток, механику и скорость. |
| `SFILT` | Переключатель | Фильтр StallGuard. Уменьшает шум показаний, но делает реакцию менее резкой. |
| `TCOOLTHRS` | `0..1048575` | Сырой порог `TSTEP` для диапазона работы StallGuard/CoolStep. Значение зависит от скорости, микрошагов и частоты STEP. |
| `THIGH` | `0..1048575` | Верхний скоростной порог в сырых единицах `TSTEP`. Связан с высокоскоростными режимами и ограничением CoolStep/StallGuard. |
| `SEMIN` | `0..15` | Нижний порог CoolStep. `0` обычно отключает CoolStep. |
| `SEMAX` | `0..15` | Верхний порог CoolStep. Задает область, где драйвер будет менять ток. |
| `SEUP` | `0..3` | Скорость увеличения тока CoolStep при росте нагрузки. |
| `SEDN` | `0..3` | Скорость уменьшения тока CoolStep при снижении нагрузки. |
| `SEIMIN: 1/2 или 1/4` | Переключатель | Минимальный ток CoolStep относительно `IRUN`: половина или четверть. |
| `SG: ...` | Только чтение | `SG_RESULT`, флаг stall и флаг standstill. Полезно смотреть при нагрузке на вал. |
| `Сохранить` | Кнопка | Сохраняет TMC-настройки. |

## TMC2160: `Диагностика`

| Поле | Что показывает |
| --- | --- |
| `DRV_STATUS` | Сырой регистр состояния драйвера. Используется как полная диагностическая маска. |
| `GSTAT R/E/UV` | `reset / drv_err / uv_cp`: сброс драйвера, ошибка драйвера, просадка charge pump. |
| `IOIN v... D0/D1` | Версия чипа и текущие уровни входов `DIAG0`/`DIAG1`. |
| `OT/OL/S2G` | Температурное предупреждение/перегрев, open-load по фазам, short-to-ground по фазам. |
| `SG: ...` | Результат StallGuard, флаг stall и standstill. |
| `CS actual` | Текущая шкала тока. |
| `PWM scale` | Текущая шкала PWM StealthChop. |
| `Очистить GSTAT` | Записывает флаги очистки в `GSTAT`, чтобы сбросить latched-ошибки. |
| `Обновить` | Перерисовать экран и перечитать диагностические значения. |

## TMC2160: `Эксперт`

Эти параметры низкоуровневые. Неверные значения могут привести к перегреву, ошибкам защиты, потере момента, шуму или некорректным DIAG-сигналам. Если нет понятной причины менять поле, лучше оставить дефолт.

| Поле | Диапазон | Что значит и на что влияет |
| --- | --- | --- |
| `Recalibrate` | Переключатель | Повторная/дополнительная калибровка внутренних аналоговых цепей драйвера. |
| `Multistep filt` | Переключатель | Фильтр STEP-входа для более стабильной оценки скорости драйвером. |
| `Stop enable` | Переключатель | Разрешение аппаратной логики stop внутри драйвера. Использовать только если понятно, какие входы/события участвуют. |
| `DIAG0 error` | Переключатель | Выводить ошибку драйвера на `DIAG0`. |
| `DIAG0 OTPW` | Переключатель | Выводить предупреждение о перегреве на `DIAG0`. |
| `DIAG0 stall` | Переключатель | Выводить stall-событие на `DIAG0`. |
| `DIAG1 stall` | Переключатель | Выводить stall-событие на `DIAG1`. |
| `DIAG1 index` | Переключатель | Выводить index-сигнал на `DIAG1`. |
| `DIAG1 onstate` | Переключатель | Выводить состояние включения/активности на `DIAG1`. |
| `DIAG1 skipped` | Переключатель | Выводить событие пропущенных шагов на `DIAG1`. |
| `DIAG0 pushpull` | Переключатель | Режим выхода `DIAG0`: push-pull вместо открытого стока/иного дефолта драйвера. |
| `DIAG1 pushpull` | Переключатель | Режим выхода `DIAG1`: push-pull вместо открытого стока/иного дефолта драйвера. |
| `S2VS level` | `0..15` | Порог защиты short-to-supply. Чем ниже/выше значение, тем иначе срабатывает защита короткого на питание. |
| `S2G level` | `0..15` | Порог защиты short-to-ground. |
| `Short filter` | `0..3` | Фильтрация детектора короткого замыкания. |
| `Short delay` | Переключатель | Дополнительная задержка/режим реакции защиты короткого замыкания. |
| `BBM time` | `0..24` | Break-before-make deadtime силового каскада. Влияет на переключение MOSFET-ключей. |
| `BBM clocks` | `0..15` | Альтернативная/дополнительная настройка deadtime в тактах. |
| `OT select` | `0..3` | Выбор порога температурного предупреждения/защиты. |
| `DRV strength` | `0..3` | Сила управления затворами внешних MOSFET. Влияет на скорость переключения, нагрев и помехи. |
| `ISENSE filter` | `0..3` | Фильтрация сигнала измерения тока. |
| `Сохранить` | Кнопка | Сохраняет TMC-настройки. |

## Дефолтные значения TMC2160

| Параметр | Дефолт |
| --- | --- |
| `Режим` | `SpreadCycle` |
| `Ток RMS` | `1000 мА` |
| `Микрошаг` | `1/16` |
| `MicroPlyer` | `Выкл` |
| `IRUN / IHOLD / IHOLDDELAY` | `31 / 16 / 6` |
| `GLOBAL_SCALER` | `0` |
| `TOFF / TBL / HSTRT / HEND` | `8 / 1 / 5 / 2` |
| `PWM_AUTOSCALE / PWM_AUTOGRAD` | `Вкл / Вкл` |
| `PWM_OFS / PWM_GRAD / PWM_FREQ / PWM_REG / PWM_LIM` | `30 / 0 / 0 / 4 / 12` |
| `StallGuard/CoolStep` | Выключены нейтральными нулями, пока не настроены вручную. |
| `dcStep` | Не используется на MKS TMC2160-OC V1.0: пины `DCEN/DCO` не выведены на доступные разъемы. |

## Почему dcStep отключен

На плате **MKS TMC2160-OC V1.0** доступ есть к `DIAG0` и `DIAG1`, но нет удобного доступа к `DCEN/DCO`. Без этих линий dcStep нельзя нормально включать и контролировать из прошивки.

Поэтому dcStep убран из пользовательского меню и не применяется при общей конфигурации драйвера. Для практической защиты и диагностики используем доступные линии:

| Линия | Практическое применение |
| --- | --- |
| `DIAG0` | Ошибка драйвера, предупреждение перегрева `OTPW`, stall-событие. |
| `DIAG1` | StallGuard, index/onstate или событие пропущенных шагов, если режим драйвера это поддерживает. |

## Практические советы по настройке

| Задача | Что смотреть |
| --- | --- |
| Сделать двигатель тише | `Основные -> Режим: StealthChop`, затем `StealthChop -> PWM autoscale/autograd`, при необходимости `PWM_OFS/PWM_GRAD`. |
| Увеличить момент | Проверить `Ток RMS`, затем `SpreadCycle`, затем chopper-параметры. Следить за нагревом. |
| Уменьшить нагрев на остановке | Уменьшить `IHOLD`, настроить `IHOLDDELAY`, при необходимости `Freewheel`. |
| Проверить перегрев/ошибки | `Диагностика -> OT/OL/S2G`, `GSTAT`, `DRV_STATUS`. |
| Настроить StallGuard | Под нагрузкой смотреть `SG_RESULT`, менять `SGT`, `TCOOLTHRS`, `SFILT`. |
| Настроить CoolStep | Начать с `SEMIN > 0`, затем подобрать `SEMAX`, `SEUP`, `SEDN`, смотреть `CS actual`. |
| Вернуть понятное поведение | Оставить экспертные параметры по умолчанию, `TOFF` не ставить в `0`, для событий использовать `DIAG0/DIAG1`. |
